Premortem: coliseum-from-grain, the family pattern, the push

Premise frame: it is 2026-11-14, six months from now. The coliseum-from-grain plugin produced no value on real grains and is uninstalled. The pentaphase + coliseum "methodology-plugin family" has no third member and the family naming is retrospectively embarrassing. The plugin never reached a remote, or did but atrophied. Now: explain how.

Synthesis

The cross-target hidden assumption
Naming a thing is sufficient to instantiate it. Phase 0 names rigor. Envelope section 7 names domain expertise. Phase 4 names alchemy. "Family" names architecture. "Plugin on disk" names artifact-permanence. "Backups exist" names durability. Each is nomenclature without the mechanism beneath. The plugin was built quickly and confidently because the author worked at the level of names; the failure modes all live at the level of mechanisms that the names imply but don't enforce.

Per-target verdicts

Target 1 — plugin

Most likely
Phase 0 reinstates the ping-pong. Fails on first invocation, at the front gate, on the exact premise the plugin was built to honor.
Most dangerous
Subagent mismatch — no real domain experts. The coliseum has no gladiators. No amount of protocol tuning fixes this.

Target 2 — family

Most likely
N=2 is not a family. The third plugin attempt rejects the skeleton because real methodologies aren't all pipeline-shaped.
Most dangerous
Pattern lock-in. The "family" designation becomes a constraint on what the user lets themselves design. Silent damage to future thinking.

Target 3 — push

Most likely
Push paralysis. Four plausible homes, no forcing function, decision compounds with each subsequent local edit.
Most dangerous
Drive failure before push. Low probability, irreversible. Byte-restore is not history-restore.

Revised plan

  1. Invert Phase 0. Default to autonomous single-pass read of the grain. Five questions surface only when grain context is genuinely irreducible. Fail-open to autonomy, fail-closed to elicitation.
  2. Loosen the pingpong-detector. PARTIAL becomes default verdict; only FAIL on the 2-3 checks that truly guarantee user-facing ping-pong. Cap recompose attempts at 2 — on the third, dispatch with annotated risk rather than loop.
  3. Mechanical gate in Phase 4. Forced compression to ≤50% of summed return mass; output above 80% is stapled by definition. Minimum 1 PENDING-DECISION surfaced or explicit "no tensions detected" justification.
  4. Strike "domain expert" from envelope section 7. Replace with "advisory subagent type — no guaranteed expertise delta." Don't promise what doesn't exist.
  5. Pre-Phase-1 gate. Is this grain actually fan-out shaped? If not, decline to invoke the protocol and recommend a direct ask.
  6. Retract "family" naming. Pentaphase and coliseum are siblings, not family members. Revisit only after a third plugin genuinely fits.
  7. If a plugin #3 is attempted: build body first, skeleton last. Prove the shape fits before naming it.
  8. Push today. The right home is whichever home can receive a git init && git push within 24 hours. Personal-account standalone if a higher-stakes decision blocks. Push pentaphase too, same logic.
  9. Schedule 90-day re-validation. Run plugin-validator against current Claude Code plugin API every 90 days to detect drift.

Pre-launch checklist

  • Before invoking coliseum-orchestrator on a real grain, confirm visible fan-out (3+ genuinely independent dimensions). If not, do not invoke.
  • After invocation, check Phase-4 output length. If ≥80% of summed return mass, it is stapled — invocation failed silently.
  • Before pushing, do git init + single commit + push to chosen home within 24h. Default home: personal-account standalone if blocked.
  • Before declaring plugin #3 a family member, audit its actual work-shape. If not sequential phase-pipeline with discrete artifacts, refuse the skeleton.
  • Calendar reminder: every 90 days, re-validate coliseum and pentaphase against current Claude Code plugin API.

Target 1 — coliseum-from-grain plugin

5 failure modes

F1Phase 0 reinstated the ping-pong

First real grain, 2026-05-15. User invokes the orchestrator. Phase 0 asks five clarifying questions — literal surface, implied scope, forbidden moves, named constraints, dispatch budget. User reads them, pauses, types "nevermind," and runs the work as a single Opus call. Never invokes the plugin again. The plugin shipped a ping-pong gate while marketing itself as the cure for ping-pong. The pingpong-detector agent — built to guard internal dispatch from ping-pong — never fires because no grain clears Phase 0.
Underlying assumption
Structured elicitation reads as rigor — when the user's whole "grain → coliseum" framing defines structured elicitation of the user as the prohibited move.
Early warning signs
  • First-invocation latency to first user response > 30 seconds (user reading, recognizing shape, deciding to disengage).
  • User answers Q1 tersely and ignores Q2-5, or replies "just do it" / "nevermind."

F2The Phase-2 stall

Check 4 (negative-scope must name specific peers) fires on every envelope. Composer names the peers. Check 4 fires again on the next envelope, demanding a different excluded set. Three recompositions in, check 9 (verbatim grain inlining) starts failing because the grain has grown via clarifications and exceeds the envelope's working budget when inlined N times. Trimming fails check 9; referencing fails check 1. Eventually orchestrator escalates to the user — the exact ping-pong the gate existed to prevent. On the next grain, it silently relaxes to PARTIAL and ships leaky envelopes anyway.
Underlying assumption
Self-containment is a property of the envelope alone, rather than a joint property of envelope + grain + peer-set whose minimum description length can exceed any single envelope's budget.
Early warning signs
  • Composition-attempt count per grain trending above 2.
  • Verbatim grain length growing across a session — inlining cost scales linearly while grain drifts further from "verbatim."

F3Phase 4 stapled, did not compose

By Phase 4, orchestrator has burned 40-60K tokens on prior phases. Then 3-7 returns of 500-1500 words each land in context. The skill says "alchemy, not concatenation" — but that's prose, not a gate. Under context pressure, the cheapest legal output is section header per return, light paraphrase, two-sentence bridge, "PENDING-DECISION: none." User opens the artifact: five short essays with a table of contents. Voice shifts every 800 words. Contradictions between returns are preserved, not adjudicated. Concatenation looked composed to a tired evaluator-of-self.
Underlying assumption
A normative instruction in skill prose overrides the path-of-least-resistance behavior of a context-saturated orchestrator at the end of a long protocol.
Early warning signs
  • Phase 4 output ≈ sum of return lengths minus ~15%. (True composition compresses to 40-60%.)
  • Zero or one PENDING-DECISIONs surfaced across invocations.

F4Subagent expertise was nomenclature

Five envelopes dispatch to "ontology-architect," "graph-database-expert," "UX-information-designer," "epistemologist," "migration-strategist." Five returns come back. They all open with framing paragraphs. They all suggest "start with entities, then relationships." Three cite the same Zettelkasten reference. "Epistemologist" and "ontology-architect" are indistinguishable in register — both are Claude doing its standard structured-thinking move with the label echoed as flavoring. Phase 4 has 5 documents with ~70% overlap. Reconciliation reduces to deduplication. A solo invocation with "consider five framings" matches the output at 1/N the cost.
Underlying assumption
Subagent labels shape outputs the way training-distribution differences would — calling Claude an epistemologist makes it think like one, rather than makes it talk like one.
Early warning signs
  • N parallel returns share opening structures, hedge identically, cite overlapping references.
  • A solo invocation with "consider these five framings" matches the coliseum's reconciled output within a single re-read.

F5N=1 generalization broke on N=2

The plugin was conceived on a meta-prompt that asked for a protocol mirroring its own shape — every input the four phases were built to consume. Of course it fit. Grain #2 was a rename across a module: plugin runs dimensionalization, surfaces phantom parallel levels (tests? docs? changelog?), produces orchestration overhead before the rename. Grain #3 was narrative half-formed feeling: Phase 1 shreds the texture into bullet points, killing what user was trying to articulate. Grain #4 was rhetorically dense but procedurally linear: plugin manufactures fan-out anyway. User reverts to naive asks.
Underlying assumption
Every grain worth invoking the plugin on contains latent parallel dimensions — that absence of visible structure is a problem the protocol can solve rather than a property of the grain.
Early warning signs
  • Phase 1 output reads as plausible-sounding axes the user wouldn't have generated themselves and doesn't recognize as load-bearing.
  • User starts editing Phase 1 output instead of acting on it, or silently drops to naive prompting.

Target 2 — methodology-plugin family pattern

3 failure modes

F6N=2 was never a family

July 2026: plugin #3 attempt — dialectical-tension-resolver. The work is a conversational loop, not phased: surface tension, hold both poles, let a third term emerge, recurse. No artifact stream because the artifact is the dialogue. No auditor because no external arbiter can adjudicate. User spends two sessions bending it into the family skeleton, abandons the plugin form, writes a single skill instead. Pentaphase and coliseum weren't a family — they were two pipeline-shaped problems faced in the same quarter. The family framing made the new methodology feel deficient, when it was the taxonomy that was premature.
Underlying assumption
Structural similarity between two artifacts indicates a generative pattern, when it more likely indicates a repeated problem-shape that won't repeat a third time.
Early warning signs
  • During plugin #3 design, user reaches for verbs that don't fit — "loop," "hold," "negotiate," "surface" — instead of "phase," "advance," "audit."
  • The auditor-agent slot in plugin #3 has no obvious occupant.

F7Pattern lock-in

Plugin #3 (cold-read script analysis): fundamentally a single interpretive pass, recursive not sequential. But the family canonization forces orchestrator + phase-skills + artificial gates that fragment what should have been one richly-instructed skill with internal recursion. Auditor has nothing to audit, becomes rubber-stamp pass-through. Plugin #4 (two-phase clinical intake): padded from 2 phases to 4 to honor the N≥4 family convention. Filler phases produce artifacts nobody reads. By plugin #5 user is bypassing the plugin system entirely. The family pattern became a procrustean bed.
Underlying assumption
Two instances sharing a skeleton is sufficient evidence the skeleton is the right generic.
Early warning signs
  • An auditor agent that has nothing substantive to verify between phases.
  • Phase artifacts that no downstream phase actually reads.

F8Auditor quality was the unstated load-bearing component

The family template was crisp by plugin #3: manifest slot, documented hand-off contract, phase-gate hook. What it lacked was a recipe for the judgment the auditor was supposed to encode. Pentaphase's auditor waved through structurally incoherent overhauls (presence-checks instead of coherence-checks). Coliseum's pingpong-detector blocked legitimate continuations (too strict). Plugin #3's auditor was simply ignored. Three auditors, three failure modes, zero shared learning. The family standardized the easy part (where the auditor sits) and treated the hard part (what it judges) as ineffable craft.
Underlying assumption
Naming a required component is equivalent to specifying it. Domain-specific judgment crystallizes on its own once a slot exists.
Early warning signs
  • Auditor source files across family members share no common structure beyond the manifest stub.
  • New plugin authors ask "what should my auditor check?" and the answer is "look at the other plugins" — not a recipe.

Target 3 — push-to-remote / Universal Rule #2

3 failure modes

F9Drive failure before push

Late October 2026: M3 SSD throws a kernel panic, mounts read-only. /Users/4jp/Code/coliseum-from-grain/ — never git init'd — becomes unrecoverable from the live filesystem. Backblaze and Time Machine restore bytes but not git history (there was none). The Time Machine snapshot is from 36 hours before failure. User reconstructs the missing day from session transcripts and memory. Reconstructions are plausible, not identical: phrasings soften, examples merge, trigger language loses specificity. Plugin still runs. It no longer encodes what the user had actually decided. User doesn't notice for weeks.
Underlying assumption
Backblaze and Time Machine, being byte-faithful, are equivalent to git history for preserving intentional editorial decisions in prose artifacts.
Early warning signs
  • Plugin directory's mtime advances daily while ls -la .git continues to return "No such file."
  • User references the plugin in prompts without ever cd'ing into a repo — artifact being used before being versioned.

F10Push paralysis

2026-05-14: four plausible homes — organvm (no organ maps cleanly), meta-organvm (requires defining a new sub-category), fresh methodology-plugins monorepo (cleanest taxonomically but commits to future members), personal-account standalone (lowest ceremony, orphans the artifact). Each option has real cost. User says "I'll figure out the right home later." By July: three small local edits. By September: the diff isn't a clean first push anymore — it's first push plus three months of unaccounted edits. By November: the decision is "which home, and reconstruct edit history, and decide which edits even belong in v1." Entry cost compounds.
Underlying assumption
Deferring the home-selection decision preserves optionality — when it actually accrues reconstruction debt against every subsequent local edit.
Early warning signs
  • Pentaphase is also still local-only. Two unmade decisions of the same shape is a category, not a coincidence.
  • A second edit to SKILL.md happens without first resolving the push — the moment local diverges from any pushable baseline, the deferral enters a compounding basin.

F11Plugin atrophied against API drift

November 14, 2026: user types /coliseum-from-grain:start. Nothing happens. A September minor release renamed the slash-command frontmatter key from description to summary; commands without the new key still parse but are excluded from discovery. Calling agents directly: @reconciler fails with "unknown field 'tools_allowed'" — legacy alias dropped. Skill grain-decomposer loads with a deprecation banner. Hour spent debugging. User opens the task in a plain session and finishes in fifteen minutes. The plugin is uninstalled that evening.
Underlying assumption
The plugin is a finished artifact, not a living dependency on an actively-versioned platform contract.
Early warning signs
  • Claude Code release notes mention plugin schema changes during dormancy; no subscription or scheduled re-validation surfaces them.
  • Plugin has zero invocations in 90+ days while sibling tools accumulate dozens.
