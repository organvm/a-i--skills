# Pickup Paths — where each target agent looks for relays

The `canonical-phrase.md` `<pointer-path>` slot expects an absolute path. Use
the table below to choose the right path for the target agent.

## Per-agent pickup paths

| Target agent | Pickup path pattern | Notes |
|---|---|---|
| Claude (Code) | `~/.claude/plans/{YYYY-MM-DD}-handoff-{slug}.md` | Per home CLAUDE.md plan discipline. For project-scoped relays, use `<repo-root>/.claude/plans/`. |
| Codex | `~/bound/.Codex/handoffs/{YYYY-MM-DD}-{slug}-pointer.md` | Pointer-file convention; Codex reads `handoffs/` on session start. |
| External inbound feedback (FB-*) | `~/.prompt-relay/feedback/FB-{SESSION_ID}-{TIMESTAMP}.md` | Generated by `scripts/prompt-relay.sh receive`. **Different category** — inbound feedback, not forward handoff. |
| Gemini | (constitutional gap) | Per home CLAUDE.md "Known constitutional gaps": cross-agent session lookup partial. No canonical pickup path established yet. |
| OpenCode | (constitutional gap) | Sessions live in `~/.local/share/opencode/opencode.db` — file-system pickup not yet wired. |

## Path slug conventions

- **`{YYYY-MM-DD}`** — ISO date prefix. Dated naming is mandatory (no overwrites).
- **`{slug}`** — short kebab-case description of the relay's subject.
- **`-pointer.md`** suffix on Codex paths distinguishes pointer-files from full plans.
- **`handoff-`** prefix on Claude paths distinguishes handoff-relays from regular plans.

## Choosing compression level

| Compression | Path-only suffices? | Use when |
|---|---|---|
| `minimal` | Yes — pointer path + canonical phrase | No in-flight work; context is "scope is X" |
| `medium` | No — use `standard-envelope.md` template | Standard handoff with decisions and next actions |
| `full` | No — standard-envelope + linked closeout + transcript pointer | Mid-task with substantive work-in-progress to inherit |

## When no pickup path exists

Use the no-pointer-yet variant of the canonical phrase (`Cold ground — see
<closeout-path>...` from `canonical-phrase.md`). The closeout file at the
scope serves as the implicit pickup, with the caveat that the receiver
treats it as historical context rather than live state.

## Constitutional gaps (do not paper over)

The Gemini and OpenCode rows above are explicit gaps. Per home CLAUDE.md
universal rule #1 (`N/A is a vacuum — never a resting state`), each row is
an IRF candidate. Do not invent a pickup path for these agents without
verifying the target reads from it.
