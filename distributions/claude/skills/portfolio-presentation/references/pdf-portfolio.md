# PDF Portfolio Best Practices

Guidelines for creating effective PDF portfolio documents.

## When to Use PDF Format

| Use PDF | Avoid PDF |
|---------|-----------|
| Email applications | Ongoing updates needed |
| Print-ready format | Interactive demos |
| Consistent display | Large file sharing |
| Offline viewing | Frequent iteration |
| Interview handouts | SEO/discoverability |

## Document Structure

### Page Breakdown

```
Page 1: Cover/Title
Pages 2-3: About/Introduction
Pages 4-5: Project 1 (Lead with best fit)
Pages 6-7: Project 2
Pages 8-9: Project 3
Pages 10-11: Project 4-5 (Condensed)
Page 12: Contact/CTA
```

### Optimal Length

| Audience | Pages | Project Depth |
|----------|-------|---------------|
| Recruiter screen | 6-8 | Overview only |
| Hiring manager | 10-15 | Standard case studies |
| Creative director | 15-20 | Full process |
| Client proposal | 8-12 | Results-focused |

## Image Guidelines

### Resolution Requirements

| Use Case | DPI | Dimensions |
|----------|-----|------------|
| Screen viewing | 72-150 | 1920px wide min |
| Print ready | 300 | Based on print size |
| Retina display | 144 | 2x screen dimensions |

### Image Types

```
Hero Images:
- Full-width project showcase
- High impact, polished final
- 16:9 or 4:3 aspect ratio

Process Images:
- Sketches, wireframes, iterations
- Annotated screenshots
- Before/after comparisons

Detail Images:
- Close-up interactions
- Component highlights
- Micro-animations (as stills)

Context Images:
- Real-world usage
- Device mockups
- Environmental shots
```

### File Size Optimization

Target: 5-15 MB total PDF

1. Compress images before import (80-90% JPEG quality)
2. Avoid embedded videos (use QR codes)
3. Flatten layers in design files
4. Use PDF optimizer on export

## Typography

### Font Pairing

```
Headers: Modern sans-serif
Body: Readable sans-serif or serif
Accent: Reserved for metrics/callouts

Recommended:
- Headers: Inter, Helvetica, Futura
- Body: Source Sans Pro, Georgia
- Accent: Mono font for data

Sizes:
- Project titles: 24-32pt
- Section headers: 14-18pt
- Body text: 10-12pt
- Captions: 8-10pt
```

### Hierarchy Example

```
PROJECT TITLE (24pt bold)
───────────────────────
Section Header (14pt semibold)
Body text explaining the project context and approach.
Keep paragraphs short for scannability. (10pt regular)

KEY RESULT (14pt semibold)
↑ 40% (32pt bold accent color)
```

## Color Usage

### Portfolio Color System

```
Primary: Your brand color (use sparingly)
Secondary: Complementary accent
Neutral Dark: #333333 (body text)
Neutral Light: #F5F5F5 (backgrounds)
White: #FFFFFF (page background)
```

### Color Tips

- Keep project images neutral (don't compete with work)
- Use accent color for metrics and CTAs
- Ensure 4.5:1 contrast ratio for text
- Test in grayscale for print

### Page Numbers

Position: Bottom right or center
Format: `1` or `1 / 12`
Start: After cover page (page 2 = "1")

### Clickable Elements

```
Internal Links:
- Table of contents to sections
- "Back to contents" on each page

External Links:
- Live project URLs
- LinkedIn/portfolio website
- Email (mailto:)
```

## Export Settings

### Adobe Acrobat/InDesign

```
Format: PDF/X-4 (for print) or Interactive PDF (for screen)
Compatibility: Acrobat 7 or later
Resolution: 150-300 ppi
Compression: JPEG (Medium-High)
Fonts: Embed all fonts
Color: RGB for screen, CMYK for print
Security: None (allow printing/copying)
```

### Figma Export

```
Format: PDF
Pages: All pages in order
Quality: High (100%)
Outline text: Yes (for font consistency)
```

### File Naming

```
FirstName_LastName_Portfolio_2024.pdf
FirstName_LastName_Portfolio_CompanyName.pdf
FirstName_LastName_SelectedWork.pdf
```
