# Domain Ideal-Whole Substrate

**Domain:** Project Management / Institutional Substrate
**Status:** ACTIVE
**Version:** 2.2
**Parent Skill:** Product Domain Engine (PDE)

## Purpose
This skill governs the foundational layer of any domain engagement (fitness, chess, voodoo, etc.). It answers the user's core question: *"What elements, systems, structures, environments, agents so forth make the ideal whole?"*

It dispatches an "iceberg" of agentic effort beneath the user's vision, ensuring that when the user "pierces a flag" into a domain, the substrate is ready for production.

## The Eight Strata
Every domain is composed of these eight layers:

1.  **Ontology:** Primitives, relations, and canonical vocabulary (`domain-ontology.md`).
2.  **Lineage:** Historical, current, and future timeslices across academic and industry registers (`domain-lineage.md`).
3.  **Constellation:** 75-person programmatic file of peer-level exemplars (`domain-constellation.yaml`).
4.  **Gap-map:** Market-gap analysis + heist-target list for improvements (`domain-gap-map.md`).
5.  **Practitioner Agents:** The agent fleet serving the work (historians, constellators, production-architects).
6.  **Production Stack:** Tools, surfaces, and distribution pipelines (`domain-production-stack.md`).
7.  **Internal Magnet:** Refinery rules for pulling in corpus and web material (`domain-attractor.md`).
8.  **External Contribution:** Dissertation-grade research and upstream gifts to the community (`domain-contribution-charter.md`).

## The Four Operators
Flow and recursion are governed by four load-bearing operators:

1.  **Selfish-Altruistic Loop:** Learning-for-self → Quality-of-product → Service-to-friends.
2.  **Magnetic Membrane:** Bidirectional pull (Internal Magnet) and push (External Contribution).
3.  **Portfolio Operator:** Cross-domain resonance (meta-skill extraction, domain-pair overlap, tool/audience cross-flow).
4.  **Reflexive Operator:** Every build triggers a meta-system observer to study and critique the build.

## Invocation Protocol

### Phase 0: Portfolio-Pull (Audit)
Before scaffolding, scan `~/Workspace/` for reusable engines and components.
- Output: `portfolio-reuse-map.md`.

### Phase 0.5: Engine Overlap Board
Visualize the relationship between candidate engines, overlaps, and identified gaps.
- Output: `engine-overlap-board.md`.

### Phase 1: Stratification
Instantiate the 8 layers for the target domain, leaning on Phase 0 reusables.

### Phase 2: Resonance
Fire the Portfolio and Reflexive operators to extract meta-skills and establish the meta-study layer.

## Relationship to PDE
This skill handles the **Domain** layer. The Product Domain Engine (PDE) handles the **Product** layer. PDE pulls from this substrate to instantiate specific products.

## Rules of Engagement
- **Dissertation-grade default:** Layer 8 output is substrate, not garnish.
- **Heist + Contribute:** Stealing an idea requires an equal or greater upstream gift.
- **Engine Taxonomy:** Categorize reusables as Domain, Meta, or Consultant tier.
